# 监控报表需求文档

## 一、功能概述

监控报表用于实时监控机器人运行状态和关键业务指标，支持多时间维度查看，帮助运营人员了解整体业务表现和发现问题。

## 二、时间范围

支持以下时间范围选择：
- 今日：当天 00:00 至当前时刻
- 昨日：昨天 00:00 - 24:00
- 本周：本周一 00:00 至当前时刻
- 上周：上周一 00:00 至上周日 24:00
- 本月：本月 1 日 00:00 至当前时刻
- 上月：上月 1 日 00:00 至上月最后一天 24:00

**环比计算逻辑**：
- 今日环比 = 与昨日同期对比
- 昨日环比 = 与前一日对比
- 本周环比 = 与上周同期对比
- 上周环比 = 与前一周对比
- 本月环比 = 与上月同期对比
- 上月环比 = 与前一月对比

环比公式：`(当前值 - 上期值) / 上期值 × 100%`

## 三、总览 Tab

### 3.1 核心指标卡片

| 指标名称 | 计算逻辑 | 数据来源 |
|---------|---------|---------|
| **通话总量** | 选定时间范围内发起的通话总数（包含接通和未接通） | call_records 表，status 字段所有记录 |
| **接通率** | 接通通话数 / 通话总量 × 100% | status = 'answered' 的记录数 / 总记录数 |
| **平均通话时长** | 所有接通通话的时长总和 / 接通通话数（单位：秒，显示格式：分:秒） | duration 字段，仅计算 status = 'answered' 的记录 |
| **通话时长** | 所有接通通话的时长总和（单位：分钟） | duration 字段求和，仅计算 status = 'answered' 的记录 |
| **转人工率** | 转人工通话数 / 接通通话数 × 100% | hangupReason = 'transfer' 的记录数 / status = 'answered' 的记录数 |

### 3.2 通话趋势图

- **展示内容**：按天/小时展示通话量变化曲线
- **数据维度**：
  - 今日/昨日：按小时聚合
  - 本周/上周：按天聚合
  - 本月/上月：按天聚合
- **计算逻辑**：每个时间点的通话总量（同上）

### 3.3 时段分布热力图

- **展示内容**：24 小时 × 7 天的通话量热力分布
- **计算逻辑**：统计每个时段（小时）的通话总量，用颜色深浅表示通话密集程度
- **用途**：帮助识别高峰时段，优化人力配置

### 3.4 机器人性能对比表

| 指标名称 | 计算逻辑 |
|---------|---------|
| **通话量** | 该机器人发起的通话总数 |
| **接通率** | 该机器人接通数 / 该机器人通话总量 × 100% |
| **平均时长** | 该机器人接通通话时长总和 / 接通数 |
| **满意度** | 该机器人接通通话的满意度评分平均值（1-5 分） |
| **主题准确率** | 该机器人正确识别主题的通话数 / 接通通话数 × 100% |
| **转人工率** | 该机器人转人工数 / 接通数 × 100% |

## 四、趋势分析 Tab

### 4.1 通话量趋势

- **展示内容**：时间序列折线图
- **计算逻辑**：每个时间点的通话总量

### 4.2 满意度趋势

- **展示内容**：时间序列折线图
- **计算逻辑**：每个时间点的平均满意度评分
- **公式**：该时间点所有有满意度评分的通话的 satisfaction 字段平均值

### 4.3 平均通话时长趋势

- **展示内容**：时间序列折线图
- **计算逻辑**：每个时间点的平均通话时长
- **公式**：该时间点所有接通通话的 duration 字段平均值

---

## 五、主题分析 Tab

### 5.1 主题识别准确率

- **展示内容**：横向柱状图，按主题名称展示准确率
- **计算逻辑**：
  - 单个主题准确率 = 该主题正确识别的通话数 / 该主题触发总次数 × 100%
  - 正确识别判定：intentMatched = true
- **颜色规则**：
  - ≥90%：绿色（优秀）
  - 75%-90%：黄色（良好）
  - <75%：红色（需优化）

### 5.2 主题触发统计

- **展示内容**：表格，展示 TOP 6 主题
- **字段说明**：
  - 主题名称：intentName 字段
  - 触发次数：该主题被识别的总次数
  - 平均时长：该主题相关通话的平均时长

### 5.3 未识别主题 TOP 10

- **展示内容**：未能匹配到已知主题的用户输入文本列表
- **计算逻辑**：
  - 统计 intentMatched = false 的通话
  - 按出现次数降序排列，取前 10
- **展示字段**：
  - 用户输入文本
  - 出现次数
  - 最近出现时间
- **功能**：支持一键添加到知识库

## 六、通话质量 Tab

### 6.1 通话时长分布

- **展示内容**：柱状图，按时长区间统计
- **时长区间**：
  - 0-30秒
  - 30-60秒
  - 1-3分钟
  - 3-5分钟
  - 5分钟以上
- **计算逻辑**：统计每个区间内的通话数量及占比

### 6.2 挂断原因分布

- **展示内容**：饼图
- **挂断原因分类**：
  - 正常结束：对话正常完成
  - 用户主动挂断：用户中途挂断
  - 转人工：转接人工客服
  - 超时挂断：通话超时自动结束
  - 异常中断：系统异常导致中断
- **计算逻辑**：统计每种挂断原因的通话数及占比

### 6.3 关键指标

| 指标名称 | 计算逻辑 |
|---------|---------|
| **平均等待时间** | 用户从发起通话到机器人首次响应的平均时长 |
| **平均处理时间** | 从机器人首次响应到通话结束的平均时长 |

### 6.4 挂断原因详情

- **展示内容**：卡片列表，展示每种挂断原因的数量和占比
- **颜色标识**：
  - 正常结束：绿色
  - 用户主动挂断：橙色
  - 转人工：蓝色
  - 超时挂断：黄色
  - 异常中断：红色

---

## 七、导出功能

- 支持导出当前时间范围的所有报表数据
- 导出格式：Excel / CSV
- 导出内容包含：
  - 核心指标汇总
  - 各维度明细数据
  - 趋势数据

---

## 八、数据来源说明

| 数据表 | 主要字段 | 用途 |
|-------|---------|-----|
| call_records | id, taskId, phoneNumber, startTime, duration, status, intentResult, botName, rounds, satisfaction, hangupReason, intentMatched, intentName, waitTime, botId | 通话明细数据 |
| bots | botId, botName | 机器人信息 |
| intents | intentId, intentName | 主题/意图信息 |

---

## 九、性能要求

- 数据刷新频率：实时更新，页面加载时自动获取最新数据
- 查询响应时间：≤3秒
- 大数据量处理：支持分页和聚合查询，避免全表扫描
