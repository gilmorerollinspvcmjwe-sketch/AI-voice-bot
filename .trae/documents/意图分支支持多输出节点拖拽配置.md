# 意图分支支持多输出节点拖拽配置实现计划

## 背景与目标

**当前问题：**
- 意图分支（条件分支节点）目前只能通过下拉框选择目标节点
- 无法通过拖拉拽方式直观地配置多个输出分支
- 用户体验不够友好，特别是在复杂流程场景下

**目标：**
- 支持通过拖拉拽方式从条件分支节点创建多个输出连接
- 保留现有的下拉框配置方式作为备选
- 实现类似 FlowOrchestration 组件中的可视化连线功能

## 技术方案

### 1. 数据结构改造

**文件：** `types.ts`

**改动：**
- 确保 `ConditionExpression` 接口支持分支 ID 标识
- 可能需要在 `IntentEdge` 中添加 `branchId` 字段用于关联分支

### 2. MicroFlowEditor 组件增强

**文件：** `components/bot/intent/MicroFlowEditor.tsx`

**核心改动：**

#### 2.1 添加连线交互状态
```typescript
// 连线拖拽状态
const [isDrawingEdge, setIsDrawingEdge] = useState(false);
const [edgeStartNodeId, setEdgeStartNodeId] = useState<string | null>(null);
const [edgeStartBranchId, setEdgeStartBranchId] = useState<number | null>(null);
const [mousePosition, setMousePosition] = useState({ x: 0, y: 0 });
```

#### 2.2 添加输出连接点（分支节点特有）
- 条件分支节点右侧需要显示多个输出连接点（根据 expressions 数量）
- 每个连接点对应一个分支表达式
- 连接点样式需要与普通节点区分

#### 2.3 实现拖拽连线逻辑
- `handleBranchHandleMouseDown`: 处理从分支连接点开始拖拽
- `handleCanvasMouseMove`: 处理拖拽过程中的连线绘制
- `handleCanvasMouseUp`: 处理连线释放，建立连接关系

#### 2.4 渲染动态连线
- 在 SVG 层中渲染正在拖拽的临时连线
- 渲染已建立的分支连线（带分支标签）

#### 2.5 分支管理逻辑更新
- 当从分支连接点拖拽到新节点时，自动创建或更新对应的 `ConditionExpression`
- 当删除连线时，清除对应的 `targetNodeId`

### 3. LogicConfig 组件适配

**文件：** `components/bot/intent/nodes/LogicConfig.tsx`

**改动：**
- 条件分支配置中，当通过拖拽建立了连接后，UI 需要显示连接状态
- 保留下拉框选择器，支持与拖拽方式的切换/同步
- 添加视觉反馈，显示哪些分支已连接，哪些未连接

### 4. 辅助功能

#### 4.1 删除连线
- 支持点击连线删除分支连接
- 删除后清除对应的 `targetNodeId`

#### 4.2 分支连接点管理
- 当添加新的条件表达式时，自动添加新的连接点
- 当删除条件表达式时，移除对应的连接点和连线

## 实现步骤

### 步骤 1: 类型定义完善
- 检查并确保数据结构支持分支连接标识
- 必要时添加 `branchIndex` 或 `branchId` 到相关接口

### 步骤 2: MicroFlowEditor 基础交互
- 添加连线拖拽相关的状态管理
- 实现从分支节点输出点开始的拖拽逻辑
- 实现动态连线的 SVG 渲染

### 步骤 3: 分支连接点渲染
- 改造条件分支节点的渲染逻辑
- 根据 `expressions` 数量动态生成多个输出连接点
- 实现连接点的视觉样式（hover、active 状态）

### 步骤 4: 连接建立逻辑
- 实现拖拽释放时的连接建立逻辑
- 自动更新节点的 `expressions` 配置
- 处理已存在连接的更新场景

### 步骤 5: LogicConfig 适配
- 在 LogicConfig 中显示分支连接状态
- 支持下拉框与拖拽方式的同步
- 添加未连接分支的视觉提示

### 步骤 6: 删除与清理
- 支持点击连线删除连接
- 处理节点删除时的连线清理
- 处理分支表达式删除时的连线清理

### 步骤 7: 测试与优化
- 测试添加/删除分支
- 测试拖拽连接/断开连接
- 测试与现有功能的兼容性
- 优化 UI/UX 细节

## 技术难点

1. **多连接点管理**: 条件节点需要根据 expressions 动态生成多个右侧连接点
2. **连线与数据同步**: 拖拽连线需要实时同步到节点的 config.expressions 中
3. **分支标识**: 需要确保连线能正确对应到具体的 expression 索引或 ID
4. **交互冲突**: 需要处理好节点拖拽、画布平移、连线拖拽之间的冲突

## 预期效果

- 用户可以像 FlowOrchestration 一样，从条件分支节点拖拽出多条连线到目标节点
- 每个分支表达式对应一个输出连接点
- 连线带有分支名称标签
- 保留下拉框选择作为备选方案
- 支持点击连线删除连接

## 参考实现

FlowOrchestration 组件 (`components/flow/FlowOrchestration.tsx`) 中的连线实现可作为参考：
- 使用 SVG 渲染贝塞尔曲线
- 节点带有输入/输出连接点指示器
- 支持拖拽创建新节点和连线
