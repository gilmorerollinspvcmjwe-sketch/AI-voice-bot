# PolyAI Flow 编辑器完善计划

## 背景

当前 Flow 编辑器基础框架已完成，但存在三个核心问题需要解决：
1. **连线交互不完整** - 新版 FlowCanvas 有连线绘制逻辑但缺少触发入口
2. **Flow 间跳转缺失** - 类型定义有 `jumpToFlowId` 但 UI 无配置入口
3. **版本管理缺失** - 类型定义有 `FlowVersion` 但完全未实现

---

## 一、连线交互设计

### 当前状态
- `FlowCanvas.tsx` 已有 `drawingSourceNodeId` 状态和连线绘制逻辑
- 节点右侧有输出连接点按钮（第 524-535 行）
- 连线创建逻辑完整（第 212-243 行）
- **问题**：连线创建后默认 label 为 "Next"，没有条件配置入口

### 设计方案

#### 1. 连线创建流程
```
用户操作：从节点右侧连接点拖拽 → 释放到目标节点
系统行为：
  1. 创建新边，edgeType 默认为 'normal'
  2. label 默认为 'Next'
  3. 自动选中新创建的边
  4. 右侧显示边配置面板
```

#### 2. 边配置面板（新增 FlowEdgeConfig 组件）
```
边配置面板
├── 基础信息
│   ├── 边标签（给人看）
│   └── 边描述（给模型看的路由信号）
├── 边类型
│   ├── 普通连线（normal）
│   ├── 条件分支（conditional）
│   ├── 兜底分支（fallback）
│   └── 跨 Flow 跳转（goto_flow）
├── 条件配置（仅 conditional 类型显示）
│   ├── 条件函数选择
│   ├── 条件表达式
│   └── 前置实体要求
├── 优先级
│   └── 数字输入（1-100，数字越大优先级越高）
└── 调试规则（可选）
    └── 强制走这条边（用于调试）
```

#### 3. 可视化增强
- 条件边显示不同颜色（如橙色）
- 兜底边显示虚线样式
- 跨 Flow 跳转边显示特殊图标
- 边标签居中显示，支持编辑

### 实现步骤
1. 创建 `FlowEdgeConfig.tsx` 组件
2. 在 `FlowStudio.tsx` 中集成边配置面板
3. 增强边的可视化样式（颜色、虚线、图标）
4. 添加边标签双击编辑功能

## 二、Flow 间跳转设计

### 当前状态
- `types.ts` 中 `FlowNodeData` 有 `gotoFlowId` 字段
- `FlowEdge` 有 `edgeType: 'goto_flow'` 类型
- **问题**：UI 上没有配置入口

### 设计方案

#### 1. 跳转配置位置
在节点配置面板的"高级设置"区域添加"流程动作"区块：
```
流程动作
├── 目标 Flow（下拉选择其他 Flow）
├── 跳转时机
│   ├── 步骤完成后自动跳转
│   └── 通过过渡函数触发跳转
└── 跳转备注（可选）
```

#### 2. 边级别跳转配置
当边类型为 `goto_flow` 时，显示目标 Flow 选择：
```
边配置 → 边类型选择 "跨 Flow 跳转"
├── 目标 Flow（下拉选择）
└── 跳转条件描述
```

#### 3. 可视化标识
- 跨 Flow 跳转的边显示特殊颜色（如紫色）
- 边标签显示目标 Flow 名称
- 画布上显示跳转箭头图标

### 实现步骤
1. 在 `FlowNodeConfig.tsx` 中添加目标 Flow 下拉选择
2. 在 `FlowEdgeConfig.tsx` 中添加 goto_flow 类型配置
3. 增强边的可视化样式
4. 添加 Flow 间跳转的校验（目标 Flow 必须存在）

## 三、版本管理设计

### 当前状态
- `types.ts` 中定义了 `FlowVersion` 接口
- **问题**：完全未实现

### 设计方案

#### 1. 版本数据结构
```typescript
interface FlowVersion {
  id: string;
  flowId: string;
  version: string; // 如 "v1.0.0"
  flowData: FlowDefinition; // 完整的 Flow 快照
  createdAt: number;
  createdBy: string;
  description: string;
  isPublished: boolean; // 是否已发布
  publishedAt?: number;
}
```

#### 2. 版本管理 UI
```
版本管理面板（右侧抽屉或独立页面）
├── 版本列表
│   ├── 版本号（v1.0.0, v1.1.0...）
│   ├── 创建时间
│   ├── 创建人
│   ├── 发布状态（已发布/草稿）
│   └── 操作按钮（查看、回滚、删除）
├── 创建新版本
│   ├── 版本号输入
│   ├── 版本描述
│   └── 创建按钮
└── 版本对比（可选）
    ├── 选择两个版本
    └── 显示差异（节点增减、边变化等）
```

#### 3. 版本操作流程
```
创建版本：
  1. 用户点击"创建版本"
  2. 输入版本号和描述
  3. 保存当前 Flow 快照
  4. 版本列表更新

发布版本：
  1. 用户选择草稿版本
  2. 点击"发布"
  3. 标记为已发布
  4. 记录发布时间

回滚版本：
  1. 用户选择历史版本
  2. 点击"回滚"
  3. 确认对话框
  4. 恢复 Flow 数据
  5. 自动创建新版本（回滚版本）
```

#### 4. 自动版本控制
- 每次保存 Flow 时自动创建草稿版本
- 版本号自动生成（如 v1.0.0, v1.0.1...）
- 保留最近 10 个版本

### 实现步骤
1. 创建 `FlowVersionManager.tsx` 组件
2. 在 `FlowStudio.tsx` 中添加版本管理入口
3. 实现版本创建、发布、回滚逻辑
4. 添加版本列表 UI
5. （可选）实现版本对比功能

## 五、技术细节

### 连线交互
- 使用现有的 `drawingSourceNodeId` 状态
- 连线创建后自动选中并显示配置面板
- 边配置通过 `FlowStudio.tsx` 的 `selectedEdgeId` 状态控制

### Flow 间跳转
- 目标 Flow 列表从 `FlowStudio` 的 `flows` 状态获取
- 跳转校验：目标 Flow 不能是当前 Flow
- 边类型切换时自动清空/填充相关字段

### 版本管理
- 版本数据存储在 `FlowConfig` 的 `versions` 字段（需新增）
- 使用 localStorage 或后端 API 持久化
- 版本快照使用深拷贝 `JSON.parse(JSON.stringify(flow))`
