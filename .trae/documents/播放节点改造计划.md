# 播放节点改造计划

## 目标
将意图技能的播放节点（PLAY_TTS）改为"播放录音"，配置方式改为从录音市场选择，不能直接写URL。

## 当前状态

### 现有播放节点
- 节点类型: `INTERACTION` / `play_tts`
- 节点名称: "播放 / TTS"
- 配置方式: 输入音频文件URL或文本内容
- 配置字段: `type` (tts/audio_file/ssml), `content`, `voiceOverride`

### 需要改造的内容
1. **节点名称**: "播放 / TTS" → "播放录音"
2. **配置方式**: 输入URL → 从录音市场下拉选择
3. **移除TTS功能**: 仅保留录音播放

## 详细设计方案

### 1. 数据结构更新

#### 1.1 更新 PlayNodeConfig 类型
```typescript
// types.ts
export interface PlayNodeConfig extends NodeCommonConfig {
  // 移除旧字段
  // type: 'tts' | 'audio_file' | 'ssml';  // 移除
  // content: string;  // 移除
  // voiceOverride?: string;  // 移除
  
  // 新增字段
  audioId?: string;           // 录音ID
  audioName?: string;         // 录音名称（用于显示）
  audioUrl?: string;          // 录音URL（从录音市场获取）
  onErrorNodeId?: string;     // 播放异常时跳转节点
}
```

#### 3.2 改造 InteractionConfig 组件
```typescript
// InteractionConfig.tsx

// 播放录音节点配置
if (node.subType === 'play_tts' || node.subType === 'play_audio') {
  return (
    <>
      <Label label="选择录音" required />
      <AudioSelector
        value={node.config?.audioId || ''}
        onChange={(audioId, audioName, audioUrl) => {
          onChange({ 
            audioId, 
            audioName, 
            audioUrl 
          });
        }}
        audioList={audioMarketList}  // 从录音市场获取的列表
      />
      
      {/* 显示已选录音信息 */}
      {node.config?.audioId && (
        <div className="p-3 bg-blue-50 border border-blue-100 rounded">
          <div className="text-sm font-medium text-slate-700">
            {node.config.audioName}
          </div>
          {node.config.duration && (
            <div className="text-xs text-slate-500 mt-1">
              时长: {formatDuration(node.config.duration)}
            </div>
          )}
          <button 
            onClick={() => previewAudio(node.config.audioUrl)}
            className="mt-2 text-xs text-primary hover:text-primary/80 flex items-center"
          >
            <Play size={12} className="mr-1" /> 预览播放
          </button>
        </div>
      )}
      
      {/* 异常处理 */}
      <SimpleErrorHandling
        label="播放异常时跳转至"
        tooltip="包括录音文件不存在、格式不支持等"
        value={node.config?.onErrorNodeId || ''}
        onChange={(value) => onChange({ onErrorNodeId: value })}
        availableNodes={availableNodes}
      />
    </>
  );
}
```

### 6. 注意事项

1. **向后兼容**: 旧配置中的 URL 需要提示用户重新选择录音
2. **录音市场集成**: 需要从录音市场获取录音列表
3. **预览功能**: 添加录音预览播放功能
4. **权限控制**: 确保用户有权限访问录音市场
5. **数据验证**: 确保选择的录音ID有效
