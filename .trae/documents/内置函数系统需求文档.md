# 语音 Agent 内置函数系统 - 需求文档

## 一、功能概述

为语音对话 Agent 提供一套内置函数体系，支持流程控制和业务执行两类操作，使 Agent 能够自主调用工具完成业务任务。

## 二、函数分类

### 1. 过渡函数（Transition Functions）
用于流程控制，决定对话走向，在提示词中显式引用调用。

| 函数名 | 功能描述 |
|--------|----------|
| `save_state` | 保存变量到对话状态 |
| `goto_step` | 跳转到指定步骤 |
| `goto_flow` | 跳转到另一个流程 |
| `conditional_goto` | 根据条件跳转到不同步骤 |
| `collect_value` | 收集用户输入并保存到变量 |
| `check_verification` | 检查用户验证状态，支持重试机制 |

### 2. 全局函数（Global Functions）
用于业务执行，由 LLM 自主决定何时调用，对应 PolyAI 的 Global Functions。

#### 基础业务函数
| 函数名 | 功能描述 |
|--------|----------|
| `transfer_call` | 转接到人工客服或指定技能组 |
| `confirm_reservation` | 确认预约信息 |
| `hangup` | 挂断电话 |
| `send_sms` | 发送短信通知 |
| `say` | 直接对用户说话，绕过 LLM 生成 |

#### 流程控制函数
| 函数名 | 功能描述 |
|--------|----------|
| `goto_flow` | 切换到另一个 flow |
| `exit_flow` | 退出当前 flow |
| `goto_csat_flow` | 进入满意度调查流程 |

#### 通信函数
| 函数名 | 功能描述 |
|--------|----------|
| `send_email` | 发送邮件通知 |
| `call_api` | 调用已注册的 API 操作 |

#### 语音控制函数
| 函数名 | 功能描述 |
|--------|----------|
| `set_voice` | 设置语音音色 |
| `set_language` | 设置对话语言 |
| `set_asr_biasing` | 动态设置 ASR 偏置，提高特定词汇识别率 |
| `clear_asr_biasing` | 清除动态 ASR 偏置 |

#### 系统函数
| 函数名 | 功能描述 |
|--------|----------|
| `discard_recording` | 丢弃当前通话录音（隐私保护） |
| `write_metric` | 写入自定义指标数据 |

## 三、核心特性

### 1. 函数返回值控制
函数可以返回字典，包含以下字段来控制运行时行为：

| 返回字段 | 类型 | 作用 |
|----------|------|------|
| `utterance` | string | 直接对用户说的话 |
| `content` | string | 给 LLM 的系统提示 |
| `handoff` | dict | 发起转接 |
| `hangup` | bool | 结束会话 |
| `listen` | dict | 配置下一轮监听（ASR/DTMF/barge-in） |
| `variant` | string | 切换语音变体 |

### 2. 全局函数特有属性
| 属性 | 类型 | 说明 |
|------|------|------|
| `triggerKeywords` | string[] | 触发关键词（可选） |
| `executionStrategy` | 'sync' \| 'async' | 执行策略 |
| `playFiller` | boolean | 是否播放等待音 |
| `fillerContent` | string | 等待音内容 |

### 3. 过渡函数特有属性
| 属性 | 类型 | 说明 |
|------|------|------|
| `canGotoStep` | boolean | 是否可以使用 flow.goto_step() |
| `canGotoFlow` | boolean | 是否可以使用 flow.goto_flow() |
| `canModifyState` | boolean | 是否可以修改 conv.state |

## 四、Conversation 对象 (`conv`)

会话运行时总入口，提供以下能力：

| 能力 | 说明 |
|------|------|
| `conv.state` | 会话临时状态（可读写） |
| `conv.current_flow` | 当前所在 flow |
| `conv.current_step` | 当前所在 step |
| `conv.memory` | 跨会话长期记忆 |
| `conv.entities` | 已采集的实体 |
| `conv.goto_flow()` | 切换到另一个 flow |
| `conv.exit_flow()` | 退出当前 flow |
| `conv.say()` | 直接对用户说话 |
| `conv.set_voice()` | 设置语音音色 |
| `conv.set_language()` | 设置语言 |
| `conv.set_asr_biasing()` | 动态设置 ASR 偏置 |
| `conv.call_handoff()` | 发起转人工 |
| `conv.send_sms()` | 发送短信 |
| `conv.send_email()` | 发送邮件 |
| `conv.api.<name>.<operation>()` | 调用已注册 API |
| `conv.write_metric()` | 写入自定义指标 |
| `conv.discard_recording()` | 丢弃录音 |
| `conv.goto_csat_flow()` | 进入满意度调查 |

## 五、Flow 对象 (`flow`)

流程内部控制对象：

| 方法 | 说明 |
|------|------|
| `flow.current_step` | 当前 step 名称 |
| `flow.goto_step(step_name)` | 跳转到当前 flow 内指定 step |

## 六、函数调用原则

1. **一轮只做一个动作**：文本输出或工具调用二选一，不要同时做
2. **过渡函数必须 return**：调用 `flow.goto_step()` 后必须紧跟 `return` 防止静默覆盖
3. **step 名称大小写敏感**：跳转时必须精确匹配
4. **同步函数要快**：避免阻塞对话体验
5. **异步函数配等待音**：提升用户体验
