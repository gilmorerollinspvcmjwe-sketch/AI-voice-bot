# 单个意图画布调试功能实现计划

## 背景与目标

**当前状态：**
- BotIntentConfig 组件中有一个"调试"按钮（第 223-229 行），点击后打开 IntentFlowDebugger
- IntentFlowDebugger 是一个全屏的调试器，显示画布和执行控制
- 调试器与画布是分开的两个视图

**目标：**
- 参考 Dify 的调试模式，在单个意图画布（MicroFlowEditor）中直接集成调试功能
- 移除 BotIntentConfig 中的调试按钮入口
- 在 MicroFlowEditor 画布内部添加调试控制条（类似 Dify，固定在画布底部或顶部）
- 实现画布内直接运行、暂停、单步执行等调试功能

## 技术方案

### 1. MicroFlowEditor 组件增强

**文件：** `components/bot/intent/MicroFlowEditor.tsx`

**核心改动：**

#### 1.1 添加调试模式状态
```typescript
const [isDebugMode, setIsDebugMode] = useState(false);
const [debugState, setDebugState] = useState<DebugExecutionState>('idle');
const [currentNodeId, setCurrentNodeId] = useState<string | null>(null);
const [executionHistory, setExecutionHistory] = useState<ExecutionStep[]>([]);
const [debugVariables, setDebugVariables] = useState<Record<string, any>>({});
```

#### 1.2 集成调试引擎
- 导入并使用 debugEngine.ts 中的 DebugEngine 类
- 或者复用 IntentFlowDebugger 中的执行逻辑
- 在画布中直接执行节点，实时高亮当前节点

#### 1.3 添加调试控制条（固定在画布底部）
**位置：** 画布底部中央，类似 Dify 的调试控制条

**控制按钮：**
- 开始运行 (F5): 从 START 节点开始执行
- 暂停 (F6): 暂停当前执行
- 继续 (F6): 继续执行
- 单步执行 (F10): 执行到下一个节点
- 停止 (Esc): 停止执行
- 重置：清空执行状态

**状态显示：**
- 当前执行节点指示
- 执行进度（步骤数）
- 变量快照预览

#### 1.4 节点执行状态可视化
**当前执行节点：**
- 高亮显示（蓝色/紫色光环）
- 脉冲动画效果
- 显示"执行中"标签

**已执行节点：**
- 成功：绿色边框 + 对勾图标
- 失败：红色边框 + 错误图标
- 跳过：灰色/半透明

#### 1.5 节点点击交互
- 点击已执行的节点，显示该节点的执行详情
- 显示输入/输出变量
- 显示执行时间
- 显示错误信息（如果有）

### 2. BotIntentConfig 组件简化

**文件：** `components/bot\intent\BotIntentConfig.tsx`

**改动：**
- 移除 `isDebugMode` 状态
- 移除调试按钮（第 223-229 行）
- 移除 `IntentFlowDebugger` 的导入
- 移除调试模态框相关代码

### 3. 样式与动画

**节点高亮样式：**
```css
/* 当前执行节点 */
ring-4 ring-purple-400 ring-opacity-60 animate-pulse

/* 成功执行节点 */
border-green-400 shadow-green-100

/* 失败执行节点 */
border-red-400 shadow-red-100

/* 跳过节点 */
opacity-50 grayscale
```

**控制条样式：**
- 固定在画布底部
- 半透明背景（毛玻璃效果）
- 圆角按钮
- 快捷键提示

### 4. 执行逻辑

#### 4.1 执行流程
1. 用户点击"开始"按钮
2. 查找 START 节点
3. 根据节点的 `nextNodeId` 或分支配置找到下一个节点
4. 高亮当前节点，模拟执行（或调用真实 API）
5. 更新变量
6. 重复步骤 3-5 直到流程结束

#### 4.2 分支处理
- 条件分支节点：根据当前变量值判断走哪个分支
- 支持在分支前暂停，允许用户手动选择分支（用于测试）

#### 4.3 变量管理
- 初始变量：可以从外部传入（如用户姓名、手机号等）
- 执行过程中更新变量
- 提供变量查看器面板（可选，类似 IDE 的变量窗口）

## 预期效果

### UI 布局
```
┌─────────────────────────────────────────┐
│  MicroFlowEditor 画布                    │
│                                         │
│   [START] → [节点 1] → [分支] → [节点 2]  │
│                      ↘ [节点 3]          │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │ ▶ 开始  ⏸ 暂停  ⏹ 停止  🔄 重置   │  │ ← 调试控制条
│  │ 步骤：5 | 当前：节点 2 | 变量：3   │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 交互流程
1. 用户在画布中配置好流程
2. 点击底部控制条的"开始"按钮
3. 画布中的节点依次高亮执行
4. 执行完成的节点显示状态（成功/失败）
5. 点击节点可查看执行详情
6. 支持暂停、单步执行等高级调试功能

## 技术难点

1. **执行引擎集成**: 需要将调试执行逻辑无缝集成到画布组件中
2. **状态管理**: 调试状态与画布状态的同步
3. **分支处理**: 条件分支的自动判断与手动干预
4. **性能优化**: 大量节点执行时的性能表现
5. **错误处理**: 执行错误的捕获与展示
