## 数据结构设计

```typescript
interface UnknownQuestion {
  id: string;
  question: string;                    // 用户原始问题
  context: string;                     // 上下文（如：机器人说了什么导致用户问这个）
  botResponse: string;                 // 机器人的回复
  frequency: number;                    // 出现次数
  confidence: number;                   // 置信度（0-1）
  sourceId: string;                    // 来源通话ID
  status: 'pending' | 'added' | 'ignored'; // 处理状态
  extractedTime: number;                // 提取时间
  relatedKnowledge?: string;            // 如果已有相关知识，关联的知识ID
}

interface AddToKnowledgeData {
  question: string;                    // 标准问题
  similarQuestions: string[];          // 相似问题
  answer: string;                       // 标准答案
  category: string;                     // 分类
}
```

## UI 设计要点

### 列表项展示
- 问题内容（主要信息）
- 机器人回复示例（展示为什么无法回答）
- 出现次数（热度指标）
- 来源链接
- 操作按钮

### 操作流程
1. 查看问题详情
2. 选择处理方式：
   - 添加到知识库 → 填写答案 → 保存
   - 标记为无效 → 确认 → 完成
3. 批量处理可提高效率

### 状态标识
- 待处理：橙色标识
- 已添加到知识库：绿色标识
- 已忽略：灰色标识

## 技术实现注意事项

1. **状态隔离**：两个 Tab 的筛选状态需要独立管理
2. **数据一致性**：添加知识后需要同步更新
3. **性能优化**：大量数据时需要分页和虚拟滚动
4. **用户体验**：
   - 操作反馈要清晰
   - 批量操作要有确认
   - 成功/失败提示要明确

## 预期成果

完成后，"知识发现"页面将包含两个完整的功能模块：
1. **有答案抽取**：管理从对话中提取的高质量问答对
2. **未知问题**：管理机器人无法回答的问题，并支持快速转化为知识

这将形成一个完整的知识运营闭环：
```
对话数据 → 有答案抽取 + 未知问题识别 → 人工审核 → 知识库补充 → 机器人能力提升
```
