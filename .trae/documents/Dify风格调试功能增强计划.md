# Dify 风格调试功能增强计划

## 背景与目标

**当前状态：**
- 已实现基础的节点高亮和流程执行
- 有简单的调试控制条
- 有基础的变量监视器

**目标：**
- 参考 Dify 调试界面，提供完整的执行跟踪功能
- 显示每个节点的详细执行信息（输入/输出/耗时/日志）
- 实现侧边栏显示执行轨迹
- 保持与画布的联动（高亮当前执行节点）

## 技术方案

### 1. 增强执行数据结构

**文件：** `types.ts`

**改动：**
```typescript
interface ExecutionStep {
  id: string;
  nodeId: string;
  timestamp: number;
  executionInfo: {
    nodeId: string;
    status: 'success' | 'error' | 'skipped' | 'running';
    input: Record<string, any>;
    output: Record<string, any>;
    startTime: number;
    endTime?: number;
    duration?: number;
    error?: string;
    logs: string[];  // 新增：执行日志
    attemptCount: number;
  };
  variablesSnapshot: Record<string, any>;
}

// 新增：调试会话状态
interface DebugSession {
  id: string;
  startTime: number;
  endTime?: number;
  status: 'running' | 'paused' | 'completed' | 'error';
  steps: ExecutionStep[];
  variables: Record<string, any>;
  totalDuration?: number;
}
```

### 2. MicroFlowEditor 组件增强

**文件：** `components/bot/intent/MicroFlowEditor.tsx`

#### 2.1 增强调试状态
```typescript
const [debugSession, setDebugSession] = useState<DebugSession | null>(null);
const [showDebugSidebar, setShowDebugSidebar] = useState(true);
const [selectedStepId, setSelectedStepId] = useState<string | null>(null);
```

#### 2.2 增强执行逻辑
- 为每个执行步骤添加详细信息（输入/输出/日志）
- 记录执行耗时
- 添加错误处理和日志记录

#### 2.3 新增调试侧边栏
- 位于画布右侧，可折叠
- 显示执行轨迹（按时间顺序）
- 每个步骤显示：
  - 节点名称和类型
  - 执行状态（成功/失败/运行中）
  - 执行耗时
  - 输入/输出数据
  - 执行日志
- 点击步骤可高亮对应节点并显示详细信息

#### 2.4 增强节点状态可视化
- 当前执行节点：紫色脉冲光环
- 成功节点：绿色边框 + 对勾图标
- 失败节点：红色边框 + 错误图标 + 错误消息
- 完成节点：显示执行耗时标签

### 3. 执行日志系统

#### 3.1 日志记录机制
- 在每个节点执行前后记录日志
- 包括：开始执行、参数传递、API调用、结果返回、异常等

#### 3.2 日志格式化显示
- 不同类型的日志使用不同颜色
- 错误日志高亮显示
- 支持日志折叠/展开

### 4. 侧边栏 UI 设计

#### 4.1 结构
```
┌─────────────────────────────┐
│ 执行轨迹         [×] [清空] │
├─────────────────────────────┤
│ [●] START (123ms)           │
│ ├── 输入: {...}             │
│ ├── 输出: {...}             │
│ └── 日志: 开始执行...       │
├─────────────────────────────┤
│ [✓] 问好节点 (45ms)         │
│ ├── 输入: {...}             │
│ ├── 输出: {...}             │
│ └── 日志: 发送消息...       │
├─────────────────────────────┤
│ [⚠] 条件节点 (67ms)         │
│ ├── 输入: {...}             │
│ ├── 输出: {...}             │
│ └── 日志: 条件不满足...     │
└─────────────────────────────┘
```

#### 4.2 交互
- 点击步骤高亮对应节点
- 悬停显示摘要
- 可折叠/展开详细信息
- 支持滚动到当前执行节点

### UI 布局

│                                         │                │

### 功能特性

1. **执行轨迹**: 按时间顺序显示所有执行步骤
2. **详细信息**: 每个步骤显示输入/输出/耗时/日志
3. **节点联动**: 点击步骤高亮对应节点
4. **实时日志**: 执行过程中实时显示日志
5. **错误追踪**: 详细错误信息和堆栈跟踪
6. **性能监控**: 每个节点的执行耗时统计
7. **变量追踪**: 变量变更历史和来源

## 技术难点

1. **性能优化**: 大量执行步骤的渲染性能
2. **数据同步**: 画布节点与侧边栏数据的同步
3. **日志管理**: 大量日志的存储和检索效率
4. **UI响应**: 复杂数据结构的高效渲染
