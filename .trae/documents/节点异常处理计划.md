# 节点异常处理计划

## 目标
为流程节点添加简化的异常处理机制，每个节点只配置一个异常入口。

## 简化后的异常处理设计

### 1. 开始节点 (START)
**异常处理:** 不需要

---

### 2. 大模型节点 (LLM)
**异常场景:** API调用失败、超时、空响应、格式错误等
**异常配置:** 一个统一的异常入口
```typescript
interface LLMNodeConfig {
  // ... 其他配置
  onErrorNodeId?: string;  // 异常时跳转节点
}
```

---

### 3. 播放节点 (PLAY_TTS / PLAY_AUDIO)
**异常场景:** TTS合成失败、音频文件不存在、格式不支持、播放设备错误等
**异常配置:** 一个统一的异常入口
```typescript
interface PlayNodeConfig {
  // ... 其他配置
  onErrorNodeId?: string;  // 播放异常时跳转节点
}
```

---

### 4. 收集节点 (COLLECT)
**异常场景:** 用户输入超时、语音识别失败、用户打断等
**异常配置:** 只配置收集异常
```typescript
interface CollectNodeConfig {
  // ... 其他配置
  onCollectErrorNodeId?: string;  // 收集异常时跳转节点
}
```

---

### 5. 变量操作节点 (SET_VARIABLE)
**异常场景:** 变量不存在、提取失败、类型错误等
**异常配置:** 只配置提取失败
```typescript
interface SetVariableNodeConfig {
  // ... 其他配置
  onExtractionErrorNodeId?: string;  // 提取失败时跳转节点
}
```

---

### 6. 条件分支节点 (CONDITION)
**异常场景:** 表达式语法错误、变量访问错误
**异常配置:** 不需要（已有"均不满足"配置）

---

### 7. HTTP请求节点 (HTTP_REQUEST)
**已有异常处理:**
- ✅ 调用成功
- ✅ 调用失败
- ✅ 调用超时
- ✅ 未调用
**无需修改**

---

### 8. 转接节点 (TRANSFER)
**异常场景:** 目标忙、转接超时、目标不存在、转接被拒绝等
**异常配置:** 一个统一的转接失败入口
```typescript
interface TransferNodeConfig {
  // ... 其他配置
  onTransferErrorNodeId?: string;  // 转接失败时跳转节点
}
```

---

### 9. 短信节点 (SMS)
**异常场景:** 发送失败、手机号格式错误、模板错误等
**异常配置:** 一个失败入口（非必填）
```typescript
interface SMSNodeConfig {
  // ... 其他配置
  onSendErrorNodeId?: string;  // 发送失败时跳转节点（可选）
}
```

---

### 10. 标签节点 (TAG)
**异常场景:** 标签不存在、系统错误
**异常配置:** 不需要（非关键节点）

---

### 11. 脚本节点 (SCRIPT)
**异常场景:** 执行错误、超时、内存溢出
**异常配置:** 一个统一的异常入口
```typescript
interface ScriptNodeConfig {
  // ... 其他配置
  onErrorNodeId?: string;  // 执行异常时跳转节点
}
```

---

### 12. 结束节点 (END)
**异常处理:** 不需要

---

## 需要添加异常处理的节点汇总

| 节点类型 | 异常字段名 | 是否必填 |
|---------|-----------|---------|
| LLM | `onErrorNodeId` | 否 |
| PLAY_TTS / PLAY_AUDIO | `onErrorNodeId` | 否 |
| COLLECT | `onCollectErrorNodeId` | 否 |
| SET_VARIABLE (提取模式) | `onExtractionErrorNodeId` | 否 |
| TRANSFER | `onTransferErrorNodeId` | 否 |
| SMS | `onSendErrorNodeId` | 否 |
| SCRIPT | `onErrorNodeId` | 否 |

## UI设计

统一的异常处理配置UI：

```
┌─────────────────────────────────────────┐
│  异常处理                               │
├─────────────────────────────────────────┤
│  节点执行异常时跳转至:                  │
│  ┌───────────────────────────────┐      │
│  │ -- 选择跳转节点          ▼    │      │
│  └───────────────────────────────┘      │
│  注：异常包括xxx、xxx等情况             │
└─────────────────────────────────────────┘
```

不同节点的提示文本：
- **LLM节点**: "注：异常包括API调用失败、响应超时、返回空内容等情况"
- **播放节点**: "注：异常包括TTS合成失败、音频文件不存在、格式不支持等情况"
- **收集节点**: "注：收集异常包括用户未响应超时、语音识别失败、用户打断等情况"
- **变量节点**: "注：提取失败包括正则无匹配、大模型提取失败等情况"
- **转接节点**: "注：转接失败包括目标坐席忙、转接超时、转接被拒绝等情况"
- **短信节点**: "注：发送失败包括手机号格式错误、短信服务商错误等情况"
- **脚本节点**: "注：执行异常包括代码错误、执行超时、内存溢出等情况"

## 实现步骤

### Step 1: 更新类型定义 (types.ts)
为需要异常处理的节点类型添加异常字段

### Step 2: 创建通用异常处理配置组件
创建 `SimpleErrorHandling.tsx` 组件：
- 接收异常标签文本（如"节点执行异常"、"收集异常"等）
- 接收提示文本
- 接收节点列表和当前值
- 接收onChange回调

### Step 3: 更新各节点配置组件
在以下组件中集成异常处理配置：
- `CognitiveConfig.tsx` - LLM节点、COLLECT节点
- `InteractionConfig.tsx` - PLAY_TTS、PLAY_AUDIO节点
- `LogicConfig.tsx` - SET_VARIABLE节点（仅在提取赋值模式下显示）
- `DataConfig.tsx` - TRANSFER节点、SMS节点
- `ScriptConfig.tsx` - SCRIPT节点

## 注意事项

1. **向后兼容**: 无异常配置的节点默认行为不变（继续执行或结束）
2. **可选配置**: 所有异常处理都是可选的
3. **统一组件**: 使用同一个SimpleErrorHandling组件，通过props传入不同文本
4. **条件显示**: SET_VARIABLE节点只在提取赋值模式下显示提取失败配置
