# 提示词编辑器与引用功能 PRD

## 一、斜杠快捷引用功能

### 1.1 功能概述
在提示词编辑器中，用户输入 `/` 可快速插入变量、工具或代码块引用，实现所见即所得的编辑体验。

### 1.2 交互流程

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 在编辑器中输入 `/` | 弹出引用选择下拉菜单，菜单位置跟随光标 |
| 2 | 输入关键词搜索 | 实时过滤匹配的变量/工具/代码块 |
| 3 | 点击或回车选择 | 将选中项插入到光标位置，以彩色标签形式展示 |
| 4 | 点击标签上的 × | 删除该引用项 |

### 1.3 引用项展示样式

| 类型 | 格式 | 样式 |
|------|------|------|
| 变量 | `{{变量名}}` | 绿色标签 + Variable 图标 |
| 工具 | `/工具名` | 主色标签 + Wrench 图标 |
| 代码块 | `/代码块名` | 蓝色标签 + Code 图标 |

### 1.4 快捷键

| 按键 | 功能 |
|------|------|
| `/` | 打开引用菜单 |
| `↑` `↓` | 上下选择 |
| `Enter` | 确认选择 |
| `Esc` | 关闭菜单 |

## 二、QA知识引用工具功能

### 2.1 功能概述
在QA知识管理中，允许为每条问答绑定工具，当机器人匹配到该问答时，可调用绑定的工具获取数据或执行操作。

### 2.2 操作入口
QA管理 → 新建/编辑问答 → 表单视图 → 「绑定工具」区域

### 2.3 功能说明

| 功能点 | 说明 |
|--------|------|
| 工具选择 | 显示系统已配置的工具列表，支持多选 |
| 已选展示 | 以标签形式展示已绑定的工具，点击 × 可移除 |
| 数据保存 | 绑定的工具ID保存到 QA 对象的 `toolIds` 字段 |

### 2.4 使用场景示例
- 用户问「我的订单状态」→ 匹配QA → 调用「查询订单」工具 → 返回实时订单信息
- 用户问「附近门店」→ 匹配QA → 调用「门店查询」工具 → 返回门店列表

## 三、代码块管理功能

### 3.1 功能概述
代码块是预定义的Python代码片段，用于在对话流程中执行特定逻辑，如状态保存、流程跳转、挂机等。

### 3.2 入口位置
核心业务导航 → 代码块

### 3.3 代码块类型

| 类型 | 说明 | 是否可编辑 |
|------|------|------------|
| 内置代码块 | 系统预置，提供基础能力 | 不可编辑 |
| 自定义代码块 | 用户创建，满足特定需求 | 可编辑 |

### 3.4 内置代码块列表

| 名称 | 功能描述 |
|------|----------|
| `save_state` | 保存变量到对话状态 |
| `goto_step` | 跳转到指定流程步骤 |
| `collect_value` | 收集用户输入值 |
| `transfer_call` | 转接人工客服 |
| `confirm_reservation` | 确认预约信息 |
| `hangup` | 挂断电话 |
| `check_verification` | 检查验证码 |

### 3.5 自定义代码块

| 字段 | 说明 |
|------|------|
| 名称 | 代码块标识，用于引用 |
| 描述 | 功能说明 |
| 作用域 | flow（流程级）/ global（全局） |
| 参数 | 定义入参名称和类型 |
| 代码 | Python代码实现 |

### 3.6 代码上下文对象
代码块可访问以下对象：

| 对象 | 说明 |
|------|------|
| `conv` | 对话对象，包含状态、变量等 |
| `flow` | 流程对象，控制跳转、步骤等 |

## 四、意图节点调用工具与代码块

### 4.1 功能概述
在意图技能的LLM节点中，可绑定工具和代码块，让大模型在处理该节点时能够调用外部能力。

### 4.2 配置入口
意图技能编辑 → LLM节点配置面板 → 工具绑定 / 代码块绑定

### 4.3 工具绑定

| 功能点 | 说明 |
|--------|------|
| 工具列表 | 显示系统已配置的所有工具 |
| 多选支持 | 可同时绑定多个工具 |
| 已选展示 | 以标签形式展示，支持移除 |

### 4.4 代码块绑定

| 功能点 | 说明 |
|--------|------|
| 代码块列表 | 显示内置 + 自定义代码块 |
| 多选支持 | 可同时绑定多个代码块 |
| 代码预览 | 展示已绑定代码块的源代码 |
| 提示词引用 | 在提示词中通过 `/代码块名` 引用 |

### 4.5 执行逻辑

```
用户输入 → 意图匹配 → LLM节点
                          ↓
                    判断是否需要调用工具/代码块
                          ↓
                    执行调用 → 返回结果
                          ↓
                    生成回复 → 输出给用户
```

## 六、数据结构

### 6.1 QA问答
```typescript
interface QAPair {
  id: string;
  question: string;
  answer: string;
  toolIds?: string[];  // 绑定的工具ID列表
}
```

### 6.2 LLM节点配置
```typescript
interface LLMNodeConfig {
  systemPrompt: string;
  toolIds?: string[];      // 绑定的工具ID列表
  functionIds?: string[];  // 绑定的代码块ID列表
}
```

### 6.3 代码块
```typescript
interface FlowFunction {
  id: string;
  name: string;
  description: string;
  parameters: FlowFunctionParameter[];
  code?: string;
  scope: 'flow' | 'global';
  isBuiltIn: boolean;
}
```
