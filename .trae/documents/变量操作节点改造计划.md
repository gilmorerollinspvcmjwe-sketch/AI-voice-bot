# 变量操作节点改造计划

## 目标
改造现有的变量操作节点（set_variable），保留原有的系统赋值和加减操作，增加语音提取赋值功能，支持正则提取和大模型提取两种方式。

## 当前状态
- 节点类型: `LOGIC` / `set_variable`
- 当前配置: `operations` 数组，支持 `SET`/`ADD`/`SUBTRACT` 操作
- 配置界面: LogicConfig.tsx 中的简单表单

## 改造方案

### 1. 类型定义扩展 (types.ts)

```typescript
// 变量操作类型
export type VariableOperationType = 'SET' | 'ADD' | 'SUBTRACT' | 'APPEND' | 'CLEAR' | 'EXTRACT';

// 提取方式
export type ExtractionMethod = 'REGEX' | 'LLM';

// 提取规则配置
export interface ExtractionRule {
  id: string;
  targetVariable: string;      // 目标变量名
  variableType: string;        // 变量类型（从预定义类型中选择）
  method: ExtractionMethod;    // 提取方式: REGEX | LLM
  // 正则提取配置
  regexPattern?: string;       // 正则表达式
  regexGroup?: number;         // 捕获组索引（默认0）
  // 大模型提取配置（使用全局模型配置，不需要单独配置模型）
  llmPrompt?: string;          // 提取提示词
  // 通用配置
  sourceText?: string;         // 提取来源: 'lastUserInput' | 'fullContext' | 'variable:xxx'
  fallbackValue?: string;      // 提取失败时的默认值
}

// 扩展 SetVariableNodeConfig
export interface SetVariableNodeConfig extends NodeCommonConfig {
  mode: 'SYSTEM' | 'EXTRACTION';  // 系统赋值 | 提取赋值
  // 系统赋值模式（保留现有）
  operations?: VariableOperation[];
  // 提取赋值模式（新增）
  extractionRules?: ExtractionRule[];
  // 语音采集开关
  voiceCollectionEnabled?: boolean;
}
```

### 2. 组件结构

```
SetVariableConfig (重构 LogicConfig 中的 set_variable 部分)
├── ModeSwitch (模式切换: 系统赋值 / 提取赋值)
├── SystemMode (系统赋值模式 - 保留现有)
│   └── OperationList (加减赋值操作列表)
└── ExtractionMode (提取赋值模式 - 新增)
    ├── VoiceCollectionToggle (语音采集开关)
    └── ExtractionRuleList (提取规则列表)
        └── ExtractionRuleItem (单条提取规则)
            ├── VariableSelector (变量选择 - 下拉)
            ├── VariableTypeSelector (变量类型选择 - 下拉 + 添加按钮)
            ├── MethodSelector (提取方式选择: 正则 / 大模型)
            ├── RegexConfig (正则配置 - 条件渲染)
            │   ├── PatternInput (正则表达式输入)
            │   └── GroupInput (捕获组索引)
            └── LLMConfig (大模型配置 - 条件渲染)
                └── PromptTextarea (提示词输入)
```

```json
  "mode": "SYSTEM",

  ]
}
```

#### 提取赋值模式
```json
{
  "mode": "EXTRACTION",

  "extractionRules": [

      "id": "1",

      "variableType": "客户姓名",
      "method": "REGEX",
      "regexPattern": "(?:公司|单位)是(.+?)(?:的|，|$)",
      "regexGroup": 1,
      "sourceText": "lastUserInput",
      "fallbackValue": "未知公司"
    },
    {
      "id": "2",
      "targetVariable": "address",
      "variableType": "详细地址",
      "method": "LLM",
      "llmPrompt": "从以下对话中提取用户的详细地址，只返回地址文本：\n\n{{context}}",
      "sourceText": "fullContext",

    }

```

### 6. 关键交互逻辑

1. **模式切换**: 
   - 切换为 "系统赋值" 时显示操作列表
   - 切换为 "提取赋值" 时显示提取规则列表
   - 切换时保留当前配置，不丢失数据

2. **添加规则**: 
   - 点击"添加提取规则"，生成默认配置
   - 默认：变量名为空、变量类型为空、方法为REGEX

3. **变量类型管理**: 
   - 下拉选择现有类型（从全局变量类型列表获取）
   - 点击"+新增"打开弹窗创建新类型
   - 新类型自动加入全局变量类型列表并选中

4. **提取方式切换**: 
   - REGEX: 显示正则表达式输入和捕获组
   - LLM: 仅显示提示词输入（使用全局模型配置）

5. **实时预览**: 
   - 显示生成的表达式或配置摘要

### 8. 注意事项

1. **向后兼容**: 现有简单模式的配置需要正常显示和工作（默认为系统赋值模式）
2. **默认值**: 新规则添加时提供合理的默认配置
3. **验证**: 正则表达式需要验证合法性
4. **性能**: 大模型提取可能有延迟，需要异步处理提示
5. **变量类型**: 从全局变量配置中动态获取类型列表
6. **模型配置**: 大模型提取使用全局LLM配置，不需要单独配置模型
