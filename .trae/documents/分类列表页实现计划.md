# 词库管理和问答对管理分类列表页实现计划

## 目标

为词库管理和问答对管理添加一个前置的分类列表页，用户先看到所有分类的概览，点击分类后才进入该分类下的具体列表页面。

## 现状分析

### 问答对管理 (QAManager.tsx)

* 当前直接进入问答对列表，使用表格展示所有数据

* 支持按分类筛选，但分类只是筛选条件之一

* 有 RAG 配置面板在顶部

### 词库管理 (LexiconManager.tsx)

* 当前直接进入词库列表，使用表格展示所有数据

* 支持按分类筛选

* 有批量导入功能

### 分类数据

* 问答对分类：'闲聊', '业务'

* 词库分类：'产品名称', '技术术语', '行业概念', '医疗词汇', '公司名', '自定义'

## 实现方案

### 1. 问答对管理改造

#### 1.1 视图状态管理

```typescript
// 当前只有 'LIST' | 'FORM'
// 新增 'CATEGORY_LIST' 视图
type ViewType = 'CATEGORY_LIST' | 'LIST' | 'FORM';
```

#### 1.2 分类列表页设计

* **布局**：卡片网格布局，每个分类一个卡片

* **卡片内容**：

  * 分类名称（大标题）

  * 该分类下的问答对数量

  * 最后更新时间

  * 启用/禁用状态统计

  * 点击卡片进入该分类的列表页

* **操作**：

  * 新建分类按钮

  * 编辑分类名称

  * 删除分类（需确认，且分类下无数据）

#### 1.3 分类列表页组件结构

```
QAManager
├── CategoryListView (新增)
│   ├── 分类卡片网格
│   ├── 新建分类按钮
│   └── 分类操作菜单
├── QAListView (原 LIST 视图改造)
│   ├── 返回分类列表按钮
│   ├── 当前分类标题
│   └── 原列表功能
└── QAFormView (原 FORM 视图)
```

#### 1.4 数据流改造

* 进入组件默认显示分类列表

* 点击分类 → 切换到 LIST 视图，并设置当前分类筛选

* LIST 视图添加返回按钮 → 返回分类列表

### 5. UI 设计规范

#### 5.1 分类列表页

* **布局**：响应式网格，大屏 4 列，中屏 3 列，小屏 2 列

* **卡片样式**：

  * 白色背景，圆角，阴影

  * Hover 效果：轻微上浮 + 阴影加深

  * 图标：使用 Folder 或 Layers 图标

* **新建卡片**：虚线边框，+ 号图标，提示文字

#### 5.2 列表页改造

* **顶部导航**：返回按钮 + 分类标题 + 面包屑

* **筛选器**：移除分类筛选（已在分类列表选择）

* **操作按钮**：新建问答对（自动填充当前分类）

### 6. 交互流程

#### 6.1 问答对管理流程

```
进入 QAManager
    ↓
显示分类列表页（CATEGORY_LIST）
    ↓
用户选择分类（如"闲聊"）
    ↓
切换到列表页（LIST），筛选条件=闲聊
    ↓
用户可：
  - 点击返回 → 回到分类列表
  - 新建问答对 → 自动填充分类=闲聊
  - 编辑/删除问答对
```

    ↓

    ↓

### 10. 风险点

1. **数据一致性**：分类名称修改需要同步更新所有关联数据
2. **空分类处理**：删除分类前需要检查是否有数据
3. **用户体验**：频繁切换视图可能影响体验，需要平滑过渡动画

### 11. 优化建议

1. 添加分类列表页缓存
2. 分类卡片显示预览（最近更新的几条数据）
3. 支持分类拖拽排序
4. 支持分类图标自定义
