# 知识发现功能增强计划

## 目标
增强知识发现功能，支持分类选择、自动分类、批量采纳，并与问答对管理数据同步。

## 当前状态分析

### 现有知识发现功能
- 从对话中自动提取潜在问答对
- 显示提取的问题和答案
- 单个采纳功能
- 无分类管理

### 需要增强的功能
1. **分类选择/自动分类** - 采纳时可选择或修改分类
2. **批量操作** - 批量采纳、批量修改分类
3. **数据同步** - 采纳后自动同步到问答对管理

### 2. 大模型自动分类功能

#### 2.1 分类提取Prompt设计
```
请分析以下对话内容，提取问答对并自动分类。

可用分类列表：
{{categories}}

对话内容：
{{conversation}}

请输出JSON格式：
{
  "qa_pairs": [
    {
      "question": "用户问题",
      "answer": "答案内容",
      "category": "最匹配的分类名称",
      "confidence": 0.95  // 分类置信度
    }
  ]
}

注意：
1. 如果无法确定分类，category设为"未分类"
2. confidence范围0-1，表示分类确定性
3. 只从提供的分类列表中选择
```

#### 2.2 分类服务

### 4. 功能模块实现

#### 4.1 知识发现服务增强
```typescript
// services/knowledgeDiscoveryService.ts

export class KnowledgeDiscoveryService {
  // 从对话中提取问答对（带自动分类）
  async extractFromConversation(
    conversationId: string,
    options: {
      autoCategorize: boolean;
      categories: string[];
    }
  ): Promise<KnowledgeDiscoveryItem[]>;
  
  // 批量采纳
  async batchAdopt(
    itemIds: string[],
    category: string,
    options: {
      syncToQAManager: boolean;
    }
  ): Promise<BatchAdoptResult>;
  
  // 批量修改分类
  async batchUpdateCategory(
    itemIds: string[],
    category: string
  ): Promise<void>;
  
  // 同步到问答对管理
  async syncToQAManager(
    items: KnowledgeDiscoveryItem[]
  ): Promise<void>;
}
```

#### 4.2 数据同步逻辑
```typescript
// 采纳时同步到问答对管理
async function adoptAndSync(item: KnowledgeDiscoveryItem) {
  // 1. 更新知识发现条目状态
  await updateDiscoveryItem(item.id, {
    status: 'adopted',
    category: item.category,
    adoptedAt: Date.now()
  });
  
  // 2. 创建问答对
  const qaPair: QAPair = {
    id: generateId(),
    question: item.question,
    answer: item.answer,
    category: item.category || '默认分类',
    source: 'knowledge_discovery',
    sourceId: item.id,
    status: 'active',
    createdAt: Date.now()
  };
  
  // 3. 添加到问答对管理
  await qaManagerService.addQAPair(qaPair);
}
```

### 7. 注意事项

1. **分类一致性** - 知识发现的分类应与问答对管理的分类保持一致
2. **数据关联** - 采纳后保留知识发现条目与问答对的关联关系
3. **权限控制** - 批量操作需要适当的权限验证
4. **性能优化** - 批量操作考虑分批处理，避免一次性处理过多数据
5. **用户体验** - 显示分类置信度，让用户知道是自动分类还是手动指定
