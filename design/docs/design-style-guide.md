# 设计风格规范

## 1. 产品气质

VoiceAgent 是面向语音机器人配置、调试和运行排查的企业后台。界面气质应接近工业控制台：稳重、可信、低噪声、可长时间使用。当前文档面向内部，因此保留中文业务名词；视觉语言按海外 B 端产品收敛。

## 2. 视觉原则

| 原则 | 规范 |
|---|---|
| 克制 | 不使用大面积渐变、发光、营销 Hero、装饰性统计卡 |
| 清晰 | 信息层级依靠字号、字重、间距、边框和背景差异建立 |
| 高密 | 后台配置页允许中高密度，但必须保留 16–24px 的模块呼吸空间 |
| 稳定 | 常用组件状态稳定一致，危险动作必须弱默认、强确认 |
| 可追溯 | 调试、日志、外部动作、参考内容等信息应使用结构化组件表达 |

## 3. 色彩

主色使用低饱和工业蓝。成功、警告、危险、信息色只用于明确状态，不用于装饰。深色仅用于左侧导航、代码块、运行日志重点区域。

## 4. 排版

- 字体：Inter / IBM Plex Sans / 系统中文字体。
- 正文字号：14px。
- 表格和日志辅助信息：12–13px。
- 标题不使用过大字号，后台主标题建议 20–24px。
- 行高默认 1.5，长 Prompt、日志、知识片段使用 1.65–1.75。

## 5. 圆角与阴影

- 普通控件：8–10px。
- 面板：12–16px。
- 弹窗：16px。
- 阴影只用于浮层、弹窗、重点容器，不用于所有卡片。

## 6. 信息密度

- 配置页：中密度，表单行距 16–20px。
- 列表页：高密度，行高 56–64px。
- 调试页：过程信息优先，聊天、日志、外部动作和参考内容应并列可见。
- 日志弹窗：默认展示全量 Prompt，大段内容内部滚动。

## 7. 禁用表达

- 大 Hero 头图。
- 大面积炫彩渐变。
- 消费级 AI 聊天产品的强拟人视觉。
- 无意义徽章、口号、小字说明。
- 用页面级说明代替组件规格。

## 8. 交互与可访问性

- 可点击元素必须有 hover、focus、disabled 状态。
- 键盘焦点必须可见，不能只依赖鼠标 hover。
- 表单错误必须用文字说明，不只靠红色。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
- 色彩不能作为唯一状态表达，需配合文字、图标或位置。
- Prompt、JSON、日志、工具返回等可复制内容必须保留可选中文本或复制入口。

## 9. 性能与复杂数据

- 表格超过 1000 行时必须分页、虚拟滚动或服务端查询。
- Prompt 超过 1000 字时放入滚动容器，不撑高整页。
- 原始 JSON、日志详情、工具返回默认分区滚动或折叠。
- 流程画布节点超过 20 个时需要缩放、搜索或分组。
- 搜索和筛选使用防抖或显式提交。
- loading 使用局部骨架或局部 spinner，避免整页闪烁。
