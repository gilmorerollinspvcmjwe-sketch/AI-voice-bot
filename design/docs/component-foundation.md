# 基础组件规范

## 1. App Shell

| 项 | 规格 |
|---|---|
| 侧边栏宽度 | `layout.sidebarWidth` |
| 收起宽度 | `layout.sidebarCollapsedWidth` |
| 顶部栏高度 | `layout.headerHeight` |
| 内容区背景 | `color.semantic.bgPage` |
| 内容区最大宽 | 以业务密度决定，配置页建议不超过 1440px，表格页可铺满 |

App Shell 由 Sidebar、Header、MainContent 组成。Sidebar 承载业务目录，Header 承载标题、返回、全局搜索、语言和账号入口，MainContent 承载组件组合。

### 规格

| 项 | 规格 |
|---|---|
| 一级菜单高度 | 44px |
| 二级菜单高度 | 40px |
| 一级字重 | bold |
| 二级字重 | medium |
| 二级缩进 | `component.sidebar.indentLevel2` |
| 选中指示 | 左侧 3px 蓝色竖条 |
| 展开状态 | 箭头向上或旋转 180° |

## 3. Header

| 元素 | 规格 |
|---|---|
| 返回按钮 | 32–36px，图标居中 |
| 页面标题 | 20–24px，bold |
| 全局搜索 | 宽 260–320px |
| 语言选择 | 88–120px |
| 头像 | 36–40px 圆形 |

## 4. Button

| 类型 | 使用范围 | 视觉 |
|---|---|---|
| Primary | 保存、发布、新增、确认 | 主色背景，白字 |
| Secondary | 取消、返回、普通操作 | 白底、边框 |
| Text | 表格轻操作 | 透明背景，主色文字 |
| Danger | 删除、停用、清空 | 默认弱危险，确认态强危险 |
| Icon | 收起、关闭、复制、展开 | 固定正方尺寸 |

### Button 状态

default、hover、active、focus、disabled、loading、danger。

## 5. Tabs

| 类型 | 场景 |
|---|---|
| Page Tabs | 模块切换，例如基础配置、意图技能、模型调试 |
| Sub Tabs | 变量配置、业务分析中的二级切换 |
| Log Tabs | 日志弹窗中输入、Prompt、知识/工具、输出切换 |

选中态应强于 hover，但不使用高饱和大面积背景。

## 6. Badge / Tag / Status Pill

| 类型 | 场景 |
|---|---|
| Badge | 模型名、工具名、知识库名 |
| Status Pill | 启用、停用、成功、失败、待处理 |
| Tag | 业务标签、意图标签、情绪标签 |

状态色只表达状态，不作为装饰。

## 7. Card / Panel / Section

| 类型 | 场景 | 规格 |
|---|---|---|
| Panel | 大模块容器 | 白底、1px 边框、16px 圆角 |
| Card | 局部信息卡 | 白底、14px 圆角 |
| Section | 表单内部区块 | 可使用分割线或浅背景 |

Panel 和 Card 不承担页面标题职责，只承载内容分组。

## 8. Do / Don’t

| 类型 | Do | Don’t |
|---|---|---|
| 导航 | 保持一级/二级层级稳定，选中态清晰 | 为单个页面临时改变目录层级 |
| 按钮 | 主按钮克制，危险操作二次确认 | 一屏多个强主按钮，或用链接做危险操作 |
| 面板 | 承载内容分组，保持呼吸空间 | 用大卡片承载页面标题和营销文案 |
| 标签页 | 只做同一对象下的内容切换 | 用 tab 替代主导航或流程步骤 |
| 状态标签 | 表达真实状态 | 为装饰堆叠多种颜色 |
