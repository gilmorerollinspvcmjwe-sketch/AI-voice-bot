# 问答对批量操作功能产品逻辑文档

## 一、功能概述

### 1.1 功能背景

在问答对管理场景中，运营人员经常需要对大量问答对进行统一管理操作，如：
- 批量清理过期或无效的问答对
- 批量启用新导入的问答对
- 批量停用特定分类的问答对
- 批量为选中的问答对生成语音资源

### 1.2 功能目标

- 提供高效的批量操作能力，减少重复操作
- 支持灵活的选择方式（全选、单选、多选）
- 提供明确的操作确认机制，防止误操作
- 与现有批量TTS功能打通，支持对选中项生成录音

## 二、功能模块

### 2.1 批量选择机制

| 选择方式 | 触发方式 | 功能描述 |
|---------|---------|---------|
| **全选/取消全选** | 点击表头复选框 | 选中/取消选中当前列表所有问答对 |
| **单选** | 点击行内复选框 | 选中/取消选中单个问答对 |
| **多选** | 结合Shift键（可扩展） | 连续选中多个问答对 |

**视觉反馈**：
- 选中的行显示淡蓝色背景（bg-blue-50/30）
- 表头复选框显示当前选择状态（未选/部分选/全选）

### 2.2 批量操作工具栏

**显示时机**：当有问答对被选中时（selectedIds.size > 0）

**工具栏位置**：列表顶部过滤区域右侧

**操作按钮**：

| 按钮 | 图标 | 颜色 | 功能 |
|-----|------|------|------|
| 批量启用 | Power | 绿色 | 将选中的问答对状态设为启用 |
| 批量停用 | PowerOff | 橙色 | 将选中的问答对状态设为停用 |
| 批量删除 | Trash2 | 红色 | 删除选中的问答对 |
| 取消选择 | - | 灰色文字 | 清空所有选择 |

**显示信息**：
- 显示当前选中的数量："已选择 X 条"

### 2.3 批量操作确认机制

所有批量操作都需要二次确认，防止误操作。

#### 2.3.1 批量删除确认

**弹窗标题**：确认批量删除（红色图标）

**提示内容**：
- 主文案："您确定要删除选中的 X 条问答对吗？"
- 副文案："此操作不可撤销，删除后机器人将无法回答这些问题。"

**操作按钮**：
- 取消：关闭弹窗，保持选择状态
- 确认删除：执行删除操作，清空选择状态

#### 2.3.2 批量启用/停用确认

**弹窗标题**：
- 启用：确认批量启用（绿色图标）
- 停用：确认批量停用（橙色图标）

**提示内容**：
- 启用主文案："您确定要启用选中的 X 条问答对吗？"
- 停用主文案："您确定要停用选中的 X 条问答对吗？"
- 启用副文案："启用后，机器人将可以回答这些问题。"
- 停用副文案："停用后，机器人将不再回答这些问题，但数据仍保留。"

**操作按钮**：
- 取消：关闭弹窗，保持选择状态
- 确认启用/停用：执行状态切换，清空选择状态

## 三、交互流程

### 3.1 批量删除流程

```
用户操作：
  1. 勾选需要删除的问答对（可多选）
  2. 点击"批量删除"按钮
  3. 系统弹出确认对话框
  4. 用户确认删除
  5. 系统执行删除操作
  6. 清空选择状态
  7. 更新列表显示

异常处理：
  - 用户取消：关闭弹窗，保持选择状态
  - 未选中任何项：不显示批量操作工具栏
```

### 3.2 批量启用/停用流程

```
用户操作：
  1. 勾选需要变更状态的问答对（可多选）
  2. 点击"批量启用"或"批量停用"按钮
  3. 系统弹出确认对话框
  4. 用户确认操作
  5. 系统批量更新状态
  6. 清空选择状态
  7. 列表中显示新的状态

状态规则：
  - 启用：isActive = true
  - 停用：isActive = false
  - 已启用项再次启用：无变化
  - 已停用项再次停用：无变化
```

### 3.3 与批量TTS的联动

```
原有功能增强：
  1. 用户勾选问答对
  2. 打开"录音配置"弹窗
  3. 覆盖范围选项"仅勾选的问答"变为可用状态
  4. 显示当前选中的数量
  5. 选择该选项后，仅对选中的问答对生成录音

优先级：
  - 如果同时选择了"仅勾选的问答"和特定音色
  - 则只对选中项中不存在的音色进行生成
```

## 四、数据结构

### 4.1 状态管理

```typescript
// 批量操作相关状态
interface BatchOperationState {
  // 选中的问答对ID集合
  selectedIds: Set<string>;
  
  // 批量删除确认弹窗显示状态
  isBatchDeleteConfirmOpen: boolean;
  
  // 批量状态切换确认弹窗显示状态
  isBatchToggleConfirmOpen: boolean;
  
  // 批量状态切换动作类型
  batchToggleAction: 'enable' | 'disable';
}
```

### 4.2 操作函数

```typescript
// 切换全选/取消全选
function toggleSelectAll(): void

// 切换单个项的选中状态
function toggleSelectItem(id: string): void

// 执行批量删除
function handleBatchDelete(): void

// 执行批量状态切换
function handleBatchToggle(): void
```

## 五、边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 未选中任何项 | 不显示批量操作工具栏 |
| 选中项已被删除 | 从selectedIds中自动移除 |
| 列表为空 | 表头复选框禁用 |
| 部分选中 | 表头复选框显示"部分选中"状态 |
| 全部选中 | 表头复选框显示"全选"状态 |
| 操作过程中数据变更 | 保持当前选择状态，不受影响 |

## 六、性能考虑

1. **选择状态管理**：使用Set数据结构，O(1)时间复杂度
2. **批量更新**：一次性更新所有选中项，减少重渲染次数
3. **大数据量**：支持虚拟滚动（可扩展），避免卡顿
4. **操作反馈**：操作完成后清空选择，避免重复操作
